第1章 アルゴリズム

アルゴリズムとは
与えられた課題を解決するための処理手順
  • 料理のレシピ
  • ゲームの攻略法
アルゴリズムに必要な条件
正当性：必ず正しい答えを導き出す事を保証すること。
停止性：最終的に必ず停止すること。
構造化プログラミングの3つの構造
  • 逐次構造：処理を記述した順番に逐次実行する。
  • 選択構造：条件により実行する処理の流れを変える。
  • 反復構造：条件が成立する間、一定の処理を繰り返す。

プログラミングは上記の3つの構造の組み合わせでできてる。
どのプログラミングでも上記の構造に分類することができる。

第2章 変数と配列

データ型
  • 整数型：整数値（小数点を含まない）
  • 実数型：実数地（小数点を含む）
  • 文字型：1文字を扱う
  • 文字列型：複数の文字を扱う
  • 理論型：「真（true）」「偽（false）」を扱う

文字列型には専用メソッドがあり検索機能等がついている。
文字型はその機能はない分軽い。

「値」の表現
  • ダブルクォーテーション（”）で括る：”文字列型”
  • シングルクォーテーション（’）で括る：'文字型'
  • クォーテーション記号なし：整列型、実数型、理論型

JavaScript、PHPはクォーテーションの区別はなし（どちらも使用OK）

変数

変数の概要

  • データ（値）を入れる入れ物
  • 1つの変数には、データ（値）は1つしか入らない
  • 変数にデータ（値）を入れることを代入という
  • 変数名は一意であること

禁則ルール

  • 変数名は「数字のみ」「数字で始まる名前」は使用不可
  • 変数名にプログラム言語の命令文と同じ名前は使用不可（予約語）

HTMLのid、class、name属性値も同じ禁則ルール

記述方法

  • 変数にデータ（値）を代入する記述はイコールを使う
  • イコール自体は「等しい」ではなく、「代入」という意味
  • イコールの左辺に変数名、右辺に値を記述する
  • イコール2個で、左辺と右辺が「等しい」という意味
  • 変数に変数を代入した場合、コピー元の変数の中身が、コピー先へ代入される

A【左】変数  =  10【右】値

SAMPLE

STR = "ABC"

→変数STRに文字列"ABC"を代入する

変数の代入

  • 変数の代入は2つの方式がある
    • 【値渡し】
      変数内に値が格納される。すべての言語で使える
    • 【参照渡し】
      変数内にメモリ番地が格納される。オブジェクト指向言語で使える
  • 変数には、代入する値に対応するデータ型がある
  • 変数のデータ型は、Web系言語ははブラウザが決める。
    Web系以外の言語は人間が変数宣言で指定する。
配列

配列の概要

  • 複数の変数を一列に並べたもの
  • 配列にもデータ型がある
  • オブジェクト指向型言語では、配列には同じデータ型の値しか入らない
  • Web系言語（JS、PHP）では、配列には異なるデータ型が入る

禁則ルール

  • 配列名のルールは「一意であること」「先頭に数字でないこと」。変数と同じ。

記述方法

  • 配列にデータを追加したり、読み込んだりする場合は、配列の中の通し番号（配列要素番号）を指定して行う。
  • 2次元配列は配列要素番号を2個持ち、表（行と列）を格納できる
  • 文字列型は「文字型」の配列である。

var宣言 name配列名 = ["A"[0], "B"[1], "C"[2]]

[0][1][2]・・・の部分を配列要素番号という。

SAMPLE

var myname = ["山田", "鈴木", "田中"];
var age = [20, 40, 35];
var gender = ["男", "女", "男"];
var i = 1;
alert(myname[i]+"さん("+gender[i]+")の年齢は"+age[i]+"歳です");

【結果（アラート）】
鈴木さん（女）の年齢は40歳です

配列番号には変数を用いることが可能。

SAMPLE（2次元配列）

var meibo = [
 [1, "山田", 24, "男性", "東京"],
 [2, "鈴木", 30, "男性", "神奈川"],
 [3, "田中", 45, "女性", "埼玉"]
];
var i=2;
alert(
 "会員番号"+meibo[i][0]+
 "番の"+meibo[i][1]+
 "さん("+meibo[i][2]+
 ")は、"+meibo[i][4]+
 "県にお住まいです"
);

【結果（アラート）】
会員番号3番の田中さん(45)は、埼玉県にお住まいです

SAMPLE（文字列型）

var moji="あいうえお"
alert(moji[1]);

【結果（アラート）】
い

第3章 データ構造

データ構造とは
大量のデータを効率よく管理（登録、更新、削除、検索）する仕組み
データ構造の概念（5種類）
  • 配列
  • リスト
  • スタック
  • キュー
  • ツリー

スタック

データを登録した順番と逆順でデータを取り出す。
LIFO、FILO、先入れ後出し

例：Git、GitHubなど

キュー

データを登録した順番通りデータを取り出す。
FIFO、LILO、先入れ先出し

例：プリンターの印刷
